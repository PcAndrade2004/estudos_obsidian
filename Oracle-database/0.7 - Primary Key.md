## O que é uma chave primária 
É um capo que garante que :
* Cada linha da tabela seja **único**
* Não existe valores **NULL**
* Não existe **duplicação**

## Chaves primárias simples 
```SQL
CREATE TABLE TB_CLIENTES (
	ID_CLIENTE NUMBER PRIMARY KEY,
	NOME VARCHAR2(100)
);
```
* Podemos criar a chave primária já na criação da tabela. Adicionando após o tipo a palavra chave ``PRIMARY KEY`` 

### Criando a chave primária após a criação da tabela 
```SQL
ALTER TABLE TB_CLIENTES ADD CONSTRAINT PK_CLIENTES PRIMARY KEY (ID_CLIENTE);
```

## #Removendo chame primária (PK)

Podemos remover uma chave primária da seguinte maneira: 
```SQL
ALTER TABLE tb_aluno 
DROP CONSTRAINT tb_aluno_pk;
```

## Chave primária Composta 
<font color="#ff0000">Chave primária Composta</font> acontece quando unimos dois campos ou mais para<font color="#ff0000"> formarem, juntos, um identificador único</font>

Pense em um carrinho de compras. Você tem o `id_pedido` e o `id_produto`. 

Se definirmos `PRIMARY KEY (id_pedido, id_produto)`, o banco de dados aceita isto:

| id_pedido | id_produto | quantidade | O banco aceita?                                                |
| --------- | ---------- | ---------- | -------------------------------------------------------------- |
| 10        | A          | 2          | ✅ Sim (primeiro item do pedido 10)                             |
| **10**    | **B**      | 1          | ✅ Sim (o pedido 10 agora tem o produto B)                      |
| **11**    | **A**      | 5          | ✅ Sim (o produto A também está no pedido 11)                   |
| 10        | A          | 3          | ❌ **Bloqueado!** (Essa combinação já existe na primeira linha) |
